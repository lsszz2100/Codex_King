# AGENTS.md — TaskFlow 프로젝트 규칙 (샘플)

> 이 파일을 프로젝트 루트에 `AGENTS.md`로 두면 Codex가 작업 전에 읽고 규칙을 따릅니다.
> 본문 034·035·036번 참고. 자신의 프로젝트에 맞게 고쳐 쓰세요.

## 프로젝트 개요
- 이름: TaskFlow (1인 SaaS, AI 할일 관리)
- 스택: Python 3.12 / FastAPI / SQLite(개발)·PostgreSQL(운영) / 프런트는 정적 HTML+JS
- 목표: 작고 안전하게, 테스트로 보증되는 변경

## 명령 (이 명령들로 검증한다)
- 의존성 설치: `pip install -r requirements.txt`
- 개발 서버: `uvicorn app.main:app --reload`
- 테스트: `pytest -q`
- 린트: `ruff check .`
- 포맷: `ruff format .`

## 코딩 규칙
- 모든 공개 함수에 타입 힌트와 한 줄 docstring을 단다.
- 함수는 한 가지 일만. 새 파일보다 기존 모듈 확장을 우선한다.
- 에러 메시지·사용자 노출 문구는 한국어.
- 비밀값은 코드에 하드코딩하지 말고 환경변수(`os.environ`)로.

## 작업 방식
- 큰 작업은 작은 단계로 쪼개고, 각 단계마다 테스트를 추가/실행한다.
- 변경 후 반드시 `pytest -q`가 모두 통과해야 한다. 통과 로그를 보고한다.
- 구현 후 스스로 비판적 리뷰(보안·엣지케이스)를 하고 발견한 문제를 고친다.

## 금지 사항
- `main` 브랜치에 직접 푸시 금지 — 항상 브랜치 + PR.
- `requirements.txt` 임의 수정 금지 — 의존성 추가는 먼저 제안하고 승인받는다.
- `.env`, `secrets/`, `*.key` 등 비밀 파일을 읽거나 출력하지 않는다.
- 데이터베이스 마이그레이션·삭제성 작업은 반드시 사전 확인을 받는다.

## 디렉터리 메모
- `app/` 애플리케이션 코드 · `tests/` 테스트 · `scripts/` 운영 스크립트
- 같은 파일을 여러 작업이 동시에 편집하지 않게 영역을 나눈다(서브에이전트 사용 시).
